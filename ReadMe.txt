本连接池是参考其它github项目的，然后自己重新写了一遍，并且优化了部分代码，自己添加了一些注释和需要注意的地方。

1. 平台：
只支持Linux，不过可以自己改一下代码就能在Windows或者其它平台使用，因为我只使用了Linux特有的posix标准的锁。

2. 使用前提：
Linux系统需要下载mysql。
若下载了mysql，但找不到mysql.h，需要安装依赖：
sudo apt-get install libmysqlclient-dev

3. 使用：
1）首先在数据库创建test数据库。
2）cd sql/  //去到本项目的sql目录
3）./import.sh
4）make
5）./test

4. 优化相关
MySQLConnPool目前需要优化的地方：
    1. 返回值和一些类型转换相关的需要进行部分优化，例如例如数据库返回的id是uint_64，若用32位去接就会数据丢失。
    其它目前未发现有重大bug和需要优化的地方.

建议：
    连接池一般需要配合线程池使用，这两个池子的最大数量一般是一致的，这样才能最大的保持并发优势。
